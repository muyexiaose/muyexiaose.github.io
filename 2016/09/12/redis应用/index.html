redis安装

window下

1.客户端[下载](https://github.com/dmajkic/redis/downloads)
2.将服务端 redis-server.exe 打开，将客户端打开 redis-cli.exe 打开
3.关联php 将    php_redis.dll 复制到php安装目录的ext文件下，
修改php.ini配置文件，增加    extension=php_redis.dll ，重启服务器。
4.php对redis的基本操作
    //得到redis对象
    $redis = new Redis();
    //连接本地的redis服务
    $redis->connect('127.0.0.1', 6379);
    //选择redis数据库
    $redis->select(1);
    //设置redis字符串数据
    $redis->set('name', 'jack');
    //获取数据
    $name = $redis->get('name');

linux下

1.如果电脑有网，可以直接执行次命令进行下载
    wget http://download.redis.io/releases/redis-3.0.2.tar.gz -O redis-3.0.2.tar.gz
2.下载后解压tar zxvf redis-3.0.2.tar.gz到任意目录
    解压 tar -zxvf redis-3.0.2.tar.gz
3.进入redis目录  [root@localhost ~]# cd redis-3.0.2
4.编译    [root@localhost redis-3.0.2]# make
5.选择目录安装    [root@localhost redis-3.0.2]# make PREFIX=/usr/local/redis install
6.查看安装后的文件
    ll /usr/local/redis/bin/
    redis-benchmark                   Redis调试
    redis-check-aof                    Redis数据导入
     redis-check-dump                  Redis数据导出
    redis-cli                           Redis客户端
    redis-sentinel -> redis-server  Redis监测功能 （redis3.0.0后才有）
    redis-server                        Redis服务器端
7.创建配置文件目录    [root@localhost redis-3.0.2]# mkdir /usr/local/redis/etc
8.把配置文件放到etc下(配置文件在解压目录下)    [root@localhost redis-3.0.2]# cp redis.conf /usr/local/redis/etc/
9.到redis目录下     [root@localhost redis-3.0.2]# cd /usr/local/redis/
10.开启服务         [root@localhost redis]# ./bin/redis-server


这样启动的话，redis会在我们的前台显示，所以做其他的操作就不好操作，所以我们要去修改其配置文件
11.[root@localhost redis]# vim etc/redis.conf
    daemonize no   这里把no改成yes
    daemonize yes   意思是后台启动
12.再重新启动        [root@localhost redis]# ./bin/redis-server ./etc/redis.conf
13.查看redis是否已经在后台启用        pstree
14.登录redis        [root@localhost redis]# ./bin/redis-cli
15. 停止redis
    考虑到Redis有可能正在将内存中的数据同步到硬盘中，强行终止Redis进程可能会导致数据丢失。
    正确停止Redis的方式应该是向Redis发送SHUTDOWN命令：
    [root@localhost redis]# pkill -9 redis
    或者在redis命令中：
    redis>shutdown

linux下的lamp环境安装

1. wget https://github.com/nicolasff/phpredis/archive/master.tar.gz
2. 或者使用window下载好源码包，然后上传到linux去 
    https://github.com/nicolasff/phpredis
3. 解压        unzip
4. 切换到phpredis-master目录下执行导入新模块
    /usr/local/php/bin/phpize
5. 将phpredis加入php扩展模块
    a)  ./configure  --with-php-config=/usr/local/php/bin/php-config
    b) make  && make install
6. 配置php.ini
    vim /usr/local/php/etc/php.ini   #路径根据具体地环境而定
    在php.ini加入extension=”redis.so”
7. 重启服务器
    /usr/local/apache2/bin/apachectl stop
    /usr/local/apache2/bin/apachectl start

redis应用

为什么会有nosql

1.网络应用程序的规模日渐变大，我们需要存储更多的数据、服务更多的用户以及需要更强的计算能力;
2.关系型数据库是一个通用型数据库，并不适合所有的用途，它不擅长处理大量数据的写入处理、高并发请求等等一些场景。（比如写一个共同关注功能（社交网SNS），用于计算两个用户关注了哪些共同的用户） Redis可以直接利用集合（交集、差集、并集）操作中交集;
3.MySQL的扩展性差（需要复杂的技术来实现），大数据下IO压力大，表结构更改困难。
总的来说NoSQL之所以生存并得到发展，是因为它做到了传统关系型数据库做不到的事情！

nosql的优势

  • 易扩展:

    NoSQL去掉了关系数据库的关系型特性。数据之间无关系，这样就非常容易扩展。

  • 擅长处理大数据量的读写:

    NoSQL数据库都具有非常高的读写性能，尤其在大数据量下。这是因为它的无关系性，数据库的结构简单。

  • 灵活的数据模型:

    NoSQL无需事先为要存储的数据建立字段，随时可以存储自定义的数据格式。而在关系数据库里，增删字段是一件麻烦的事情。

redis介绍

redis是一个key-value存储系统。它支持存储的类型包括string(字符串)、list(链表，数据结构)、set(集合)、zset(sorted set –有序集合)和hash（哈希类型）。
Redis数据都是缓存在内存中。redis会周期性的把更新的数据写入硬盘或者把修改操作写入追加到记录文件

Key-Value存储系统简介

Key-Value Store更加注重对海量数据存取的性能、分布式、扩展性支持上，并不需要传统关系数据库的一些特性，
例如：事务、完整SQL查询支持等，因此在分布式环境下的性能相对于传统的关系数据库有较大的提升。

为什么选择Key-Value存储系统

  • 大规模的互联网应用

    对于google,阿里这样的互联网企业，每时每刻都有无数的用户在使用它们提供的互联网服务，这些服务带来的就是大量的数据吞吐量，在同一时间，会并发的有成千上万的连接数据库进行操作。在这种情况下，单台服务器或者几台服务器远远不能满足这些数据处理的需求，简单的升级服务器性能的scale up方式是不行，所以唯一可以采用的办法就是scale out(横向扩展)。scale out的方法有很多种，但大致分为两类：一类是采用RDBMS(关系型数据库管理系统)，然后 通过对数据库的垂直和水平切割将整个数据库部署到一个集群，但是这样的缺点是这样的数据是针对特定的应用，就是说，不同的应用，切割方法不一样。还有一类就是google采用的方法，抛弃RDBMS，采用key-value形式的存储，这样可以极大的增强系统的可扩展性，如果要处理的数据持续增大，多加机器就可以了。

  • key-value支持更多的并发查询

    对于RDBMS（关系型数据库），一般几百个并发的查询就可以让它很吃力了，而一个key-value Store可以很轻松的支持上千的并发查询。

Redis适用具体场景xxxx

  1. 取最新N个数据的操作
  2. 比如取网站最新文章 (list)
  3. 排行榜应用。取Top N操作
  4. 需要精确设置过期时间的应用
  5. Uniq操作，获取某段时间所有数据排重值
  6. Redis的Pub/Sub系统可以构建实时的消息系统，有很多实时聊天系统使用它做的
  7. 构建队列系统 相对于Memcached，数据结构更多样化

redis操作

String类型与操作

String是最基本的数据类型，一个键(Key)对应一个值(Value)。它能存储任何形式的字符串，包括二进制数据。你可以用String存储用户的邮箱、图片等。一个字符串类型允许存储的数据的最大容量是512MB

  1. SET key value (设置key对应的值为value)
  2. GET key (获取key的值)
  3. SETNX key value (设置key对应的值为value，如果key存在，返回0，nx是not exists)
  4. SETEX key time value (设置key对应的值为value，并制定有效时间EX是expire的缩写)
  5. TTL key (以秒为单位，返回给定key的剩余生存时间当key不存在时，返回-2，当key存在但没有设置剩余生存时间时，返回-1否则，以秒为单位，返回key的剩余时间)
  6. setrange 键 位置 子字串
    作用：替换某字串
    键：要替换对应的键
    位置：从哪个位置开始
    子字串：要替换的新内容
    会返回一个新的字串长度
  7. MSET key val [key1 val …] (同时设置多个值，成功返回ok表示所有的值都设置了，失败返回0代表所有都设置失败)
  8. MSETNX key val [key1 val …] (同时设置多个值，成功返回1表示所有的值都设置了，失败返回0代表所有都设置失败。但是不会覆盖已经存在的key)
  9. MGET key key1 (同时获取多个键的值)
  10. GETSET key value (设置key的值，并返回key的旧值)
  11. GETRANGE key start end (返回从start到end的值（二参数是负数时取倒数）)
  12. INCR key （点赞的功能就是这样写的）
    (对key的值（key的值只能是数字）进行自增操，返回最新的值)
  13. INCRBY key increment
    (与INCR类似，INCRBY可以指定增加数值，当key不存在，会自动设置key,并默认原来的为0)
  14. DECR key (对key的值进行减操作)
  15. DECRBY key decrement (与DECR类似，减少指定值)
  16. APPEND key value (给key的键值尾部添加值)
  17. STRLEN key (获取key值的长度)
  18. DEL key (删除一个key1的值，返回1代表删除成功，返回0是失败)

Hash（散列）类型

如果要开发博客的文章列表功能，如果使用的是序列化存储到string类型，如果想要获取文章的标题，必须把整个文章数据取出来反序列化，而其中占用空间最大的文章内容却是不需要的。这样会在传输和处理时造成资源浪费。所以我们需要使用hash类型实现这个功能

  • hash是一个string类型的field和value的映射表。它的添加、删除操作时间复杂度都是0(1)（平均）。
  • Hash特别适合用于存储对象(女人)。相对于将对象的每个属性存成单个string类型。将一个对象存储在Hash类型中会占用更少的内存，并且可以更方便的存取整个对象。
  1. HSET key field value (设置key的field字段值为value，当键不存在时，HSET会自动创建)
  2. HSETNX key field value (当key的field字段存在时赋值，如果存在返回0)
  3. HMSET key field1 value1 field2 value2 (同时设置key的多个字段的值)
  4. HGET key field (获取指定的key的field的值)
  5. HINCRBY key field num (给指定的key的field字段加num，注意field的值要求是整数)
  6. HEXISTS (判断指定的key的field是否存在，如果存在返回1，否则返回0)
  7. HLEN key (返回key的field的数量)
  8. HDEL key field [field1…] (删除key的field，可以删除一个或多个字段，返回被删除字段个数)
  9. HKEYS key (返回key的所有字段名)
  10. HVALS key (返回key的所有字段的值)
  11. HGETALL key (获取key中的所有字段名和字段值)

list（列表）类型

列表类型(list)是一个链表结构，主要功能是push、pop获取一个范围的所有值等等。Redis的list类型其实就是一个每个子元素都是string类型的双向链表。我们可以通过push、pop操作从链表的头部或者尾部添加、删除元素，这样list既可以作为栈，又可以作为队列。

  • list 的优缺点
    获取越接近两端的元素获取速度就越快，也就是说即使从一个有几千万个元素的链表获取靠近头部或尾部的数据也是很快的。但是，要获取链表中间部分的数据就很慢了。

  • list 适应场景

    * 社交网站的新鲜事
* 分页功能
  1. LPUSH key value [value1…] (在list左边增加一个或多个元素，返回新的列表的长度)
  2. LPOP key (从key左边弹出一个元素，返回被弹出的元素)
  3. LRANGE key start end 返回索引从start到stop之间的所有元素（包括start、end）注意：如果start的索引位置比stop的索引位置靠后，则会返回空列表。
  4. RPUSH key value [value1…] (在list右边增加一个或多个元素，返回新的列表的长度)
  5. RPOP key (从key右边弹出一个元素，返回被弹出的元素)
  6. RPOPLPUSH sourseList destinationList (从sourseList的右边
    移除一个元素并添加到destinationList的左边)
  7. LINSERT key BEFORE|AFTER value value1 (在列表中从左到右查找value元素，然后根据第二个参数是BEFORE还是AFTER来决定将value1 插入到value的前面还是后面)
  8. LSET key index value (将索引为index的元素赋值为value)
  9. LREM key count value (删除列表中前count个值为value的元素，返回被删除的元素个数)
  10. LTRIM key start end (保留start到end的值)
  11. LLEN key (返回list的长度)
